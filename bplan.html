AI 智能招聘平台 (IntelliMatch)

交互式商业与技术规划书

本部分深入探讨项目的核心愿景、商业模式和市场进入策略。我们将定义我们的使命、对B端和C端的独特价值主张，以及我们如何解决市场冷启动的“鸡生蛋”问题。

项目使命

利用LLM技术，打破传统“关键词搜索”的壁垒，真正“理解”人才与岗位的深层匹配度，提供精准、高效、可解释的匹配体验。

核心价值主张

对B端: “告别简历筛选，直达精准面试。”

对C端: “告别盲目海投，直达优质机会。”

核心竞争力

基于LLM的“简历-JD双向语义理解与精排引擎”，不仅解析事实，更理解意图、经验“含金量”和发展可能性。

商业模式与市场战略

市场定位与用户

  • 定位: AI驱动的双边招聘平台。
  • C端用户 (人才库): 早期以技术、产品、运营岗为切入点。
  • B端用户 (付费方): 早期为高成长性中小型科技企业。

GTM 市场进入战略

  1. 阶段一 (C端冷启动): 用爬虫聚合JD作诱饵，吸引C端用户上传简历，目标 10,000+ 人才数据。
  2. 阶段二 (B端验证): 凭借人才库，邀请HR免费试用，体验“岗找人”价值。
  3. 阶段三 (商业化): 启动B端付费订阅，形成B-C双边正向循环。

盈利模式 (B端为核心)

1. SaaS 订阅

基础版、专业版、企业版。提供不同级别的职位发布和人才搜索权限。

2. 按效果付费

B端在订阅基础上，按“解锁”候选人联系方式额外付费。

3. 增值服务

C端简历优化、模拟面试；B端薪酬分析、市场洞察报告。

本部分详细拆解平台为两类核心用户——求职者（C端）和企业HR（B端）所设计的功能。重点展示双方的“杀手锏”功能，即由LLM驱动的可解释性推荐。

C 端 (求职者门户)

  • 📄

    简历智能解析

    支持多格式上传，LLM自动生成结构化的、可在线编辑的个人档案。

  • 🎯

    “人找岗” - 智能推荐

    基于个人档案，自动匹配全站JD。

  • ✨

    [杀手锏] 可解释性推荐

    "为您推荐 XX 岗位 (95%)。
    理由：您的'高并发支付网关'项目经验与 JD 中的'核心交易系统'职责高度重叠..."

  • 🔔

    个人状态管理

    管理求职意向（如“积极找工作”、“观望机会”）。

B 端 (企业 HR 门户)

  • 📋

    JD 智能解析

    支持文本粘贴或文件上传，LLM自动提取关键要求。

  • 🚀

    “岗找人” - 智能推荐

    上传 JD 后，系统**实时**从“人才向量数据库”中匹配候选人。

  • ✨

    [杀手锏] 可解释性精排

    "推荐 候选人A (96%)。
    理由：该候选人 8 年 Go 经验与您 5-10 年要求契合，且 K8s 项目经验与您职责高度相关。"

  • 🔍

    AI 人才库搜索

    支持自然语言搜索（如：“找个懂K8s的Go后端”）和多维筛选。

核心数据流 (B端“岗找人”)

  1. 1

    HR 上传 JD

    请求通过 API 网关，异步调用“解析服务”。

  2. 2

    解析与存储

    LLM 解析 JD，结构化 JSON 存入 SQL，语义向量存入 Vector DB。

  3. 3

    粗排 (SQL)

    “匹配引擎”在 SQL 库中按地点、年限等硬性条件过滤。

  4. 4

    向量召回 (Vector DB)

    用“JD 向量”在 Vector DB 中召回 Top K 语义匹配的简历。

  5. 5

    LLM 精排 (Re-rank)

    将 JD 和 Top 20 简历打包发给 LLM，进行最终打分和“理由生成”。

  6. 6

    返回结果

    返回带“推荐理由”的精准列表给 HR。

实施路线图 (Gantt 视图)

各阶段关键任务

P1: 基础建设 & C 端冷启动

(Months 0-4)

  • 搭建核心技术栈 (LLM, SQL, Vector DB)
  • 开发 C 端门户 (上传简历)
  • 开发 JD 爬虫，聚合职位
  • 开发“人找岗”匹配引擎 v1
  • 目标: 向量化简历 > 10,000

P2: B 端验证 & 飞轮启动

(Months 5-9)

  • 开发 B 端门户 (上传 JD)
  • 开发“岗找人”匹配引擎及 LLM 精排
  • 邀请 50+ 种子企业 HR 免费入驻
  • 迭代匹配算法
  • 目标: 真实 HR 发布 JD > 1,000

P3: 商业化 & 规模化

(Months 10-18)

  • 上线 B 端 SaaS 订阅和支付
  • 启动 B 端市场销售团队
  • 启动 C 端付费增值服务
  • 优化 LLM 成本和系统性能
  • 目标: 付费 B 端客户 > 100
